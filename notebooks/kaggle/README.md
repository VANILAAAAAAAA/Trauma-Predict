# Kaggle Entry Points

This folder contains Kaggle-specific launch wrappers only. Core model, data, and evaluation logic belongs under `src/trauma_predict/`.

Kaggle inputs must be mounted datasets or working-directory files outside Git. Do not place MIMIC-derived data in this repository.

The completed baseline entrypoint is `train_multires_event_v1.ipynb`, pinned to tag `multires-event-v1-baseline-run-20260712-r3`. It delegates to `run_multires_event_v1.py`, requires T4 x2, prefers an attached exact `multires_event_v1_c4_full_20260712` private dataset, and otherwise uses an authenticated owner-private Kaggle CLI download fallback. The launcher downloads the outer Dataset ZIP without `--unzip`, safely extracts exactly one package layer, verifies the manifest, and normalizes the preserved `shards.zip`, `.jsonl.gz` split tree, or Kaggle-hosted `shards/<split>/*.jsonl` tree back to canonical gzip shards. It then runs exact preflight, a two-step DDP smoke, resumable 4,000-step DDP training, full validation, and export. The route is PyTorch-only: it installs `requirements-multires-kaggle.txt` and does not install Transformers or Accelerate. Attempt logs are retained under `t4x2_multires_event_v1_full/logs/attempt-NNNN/`.

The V2 route has two zero-Input two-cell Notebooks. `verify_multires_event_v2.ipynb` is the agent-owned T4 x2 gate and freezes `TRAUMA_PREDICT_V2_ACTION=verify_block`; all formal training authorization is false in its pinned source. `train_multires_event_v2.ipynb` is the prospective block-control handoff, but it is not releasable until the hosted verification output is downloaded and independently revalidated and the final r7 source gate is restored. Both clone an immutable Git tag and use Internet plus the existing Kaggle API Secrets to download and byte-verify the V1 base and r8 sidecar; neither uses a source Dataset nor any manually attached Kaggle Input. Before setup or Dataset materialization, a config-free T4 x2 process runs the exact rank-local writer/read/hash/gather path and the launcher re-opens every retained byte. The verification route then performs credential access checks, dependency/runtime validation, complete 50,350-anchor preflight, a same-process two-anchor x 100-trajectory semantic canary, two B32/GPU FP16 optimizer updates, the first 100 persisted validation anchors with 100 trajectories each, and checkpoint/resume validation. A successful verification must write `verification_complete.json` with `formal_optimizer_steps=0` and exit without a formal `SUCCESS` marker. Hosted duration projection is retained as planning evidence only and never gates a technically valid background run. The optimizer probe remains one-group AdamW over the raw 414-factor joint-NLL batch mean with no hidden `/414` scaling or gradient clipping. Nonzero ranks use a dedicated rank-local writer; deterministic rank-local failures propagate to both ranks without entering the prior 600-second cleanup wait.

Legacy Stage A uses `train_stage_a_hour.ipynb` and `run_stage_a_hour.py`; it remains available only for historical reproduction of the textual ModernBERT experiments.
