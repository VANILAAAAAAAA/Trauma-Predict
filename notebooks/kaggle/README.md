# Kaggle Entry Points

This folder contains Kaggle-specific launch wrappers only. Core model, data, and evaluation logic belongs under `src/trauma_predict/`.

Kaggle inputs must be mounted datasets or working-directory files outside Git. Do not place MIMIC-derived data in this repository.

The completed baseline entrypoint is `train_multires_event_v1.ipynb`, pinned to tag `multires-event-v1-baseline-run-20260712-r3`. It delegates to `run_multires_event_v1.py`, requires T4 x2, prefers an attached exact `multires_event_v1_c4_full_20260712` private dataset, and otherwise uses an authenticated owner-private Kaggle CLI download fallback. The launcher downloads the outer Dataset ZIP without `--unzip`, safely extracts exactly one package layer, verifies the manifest, and normalizes the preserved `shards.zip`, `.jsonl.gz` split tree, or Kaggle-hosted `shards/<split>/*.jsonl` tree back to canonical gzip shards. It then runs exact preflight, a two-step DDP smoke, resumable 4,000-step DDP training, full validation, and export. The route is PyTorch-only: it installs `requirements-multires-kaggle.txt` and does not install Transformers or Accelerate. Attempt logs are retained under `t4x2_multires_event_v1_full/logs/attempt-NNNN/`.

The V2 entrypoint is `train_multires_event_v2.ipynb`, which delegates to `run_multires_event_v2.py`. The first formal release is block-only: the Notebook defaults to immutable tag `multires-event-v2-block-run-20260713-r1`, `TRAUMA_PREDICT_V2_ACTION=block`, private target Dataset `vanilaaaa/trauma-predict-multires-event-v2-c4-r8-20260713`, and `TRAUMA_PREDICT_DRY_RUN_ONLY=0`. Source authorization accepts only `t4x2_multires_event_v2_block`; environment variables cannot authorize standalone smoke, trajectory, or relational training. The action uses one `torchrun`: an attempt-local same-process capacity gate performs exactly two B32/GPU FP16 updates plus the first 100 persisted validation anchors with 100 trajectories each, verifies memory, scaler, structural metrics, coherence, and 12-hour session headroom, then and only then permits the unchanged 4,000-step / 6,309-anchor x 100-trajectory block run. The optimizer is one-group AdamW over the raw 414-factor joint-NLL batch mean with no hidden `/414` scaling and no gradient clipping. Every step must persist complete gradient, scaler, Adam-state, LR, and resume-schedule health; `metrics.jsonl`, `optimizer_health_summary.json`, the run manifest, and `SUCCESS` are hash-bound, and incomplete or re-signed partial runs are rejected. Completed runs also persist a portable semantic runtime identity covering Python, PyTorch, CUDA, cuDNN, both GPU identities, world size, precision, and dependency-file hashes. Kaggle may expose uploaded r8 `*.jsonl.gz` target shards as plain JSONL; the launcher must reconstruct gzip with the builder-equivalent line-wise `TextIOWrapper` procedure and verify every reconstructed shard's compressed-byte SHA256 against `dataset_manifest.json`. Promotion remains a separate session requiring three explicit, distinct completed run roots; on Kaggle they must be attached inputs, share semantic runtime identity and per-sample subject identity, and the promotion path initializes no CUDA, dataset, or training process.

Legacy Stage A uses `train_stage_a_hour.ipynb` and `run_stage_a_hour.py`; it remains available only for historical reproduction of the textual ModernBERT experiments.
