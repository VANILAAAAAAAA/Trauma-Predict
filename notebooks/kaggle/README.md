# Kaggle Entry Points

This folder contains Kaggle-specific launch wrappers only. Core model, data, and evaluation logic belongs under `src/trauma_predict/`.

Kaggle inputs must be mounted datasets or working-directory files outside Git. Do not place MIMIC-derived data in this repository.

The completed baseline entrypoint is `train_multires_event_v1.ipynb`, pinned to tag `multires-event-v1-baseline-run-20260712-r3`. It delegates to `run_multires_event_v1.py`, requires T4 x2, prefers an attached exact `multires_event_v1_c4_full_20260712` private dataset, and otherwise uses an authenticated owner-private Kaggle CLI download fallback. The launcher downloads the outer Dataset ZIP without `--unzip`, safely extracts exactly one package layer, verifies the manifest, and normalizes the preserved `shards.zip`, `.jsonl.gz` split tree, or Kaggle-hosted `shards/<split>/*.jsonl` tree back to canonical gzip shards. It then runs exact preflight, a two-step DDP smoke, resumable 4,000-step DDP training, full validation, and export. The route is PyTorch-only: it installs `requirements-multires-kaggle.txt` and does not install Transformers or Accelerate. Attempt logs are retained under `t4x2_multires_event_v1_full/logs/attempt-NNNN/`.

The active V2 surface is `train_multires_event_v2_relational_primary.ipynb`. It reads one attached immutable private run bundle containing the pinned source release, train-only input normalization, and V1/r9 payload inventories. Internet is not required. To stay below Kaggle's large top-level file-list failure boundary, files no larger than 64 KiB are stored in two deterministic hash-bound tar packs; the launcher extracts only these small payloads and reports their exact count/bytes. Bulk patient shards remain directly mounted, hash-checked, and symlinked without copying or extraction. `run_relational_primary_bundle.py` requires exactly two GPUs and launches `t4x2_multires_event_v2_relational.yaml` directly. It does not clone, download, rebuild patient data, run a disposable capacity model, or silently substitute block/trajectory. Before Dataset loading, the exact production best-checkpoint save/load collective path is exercised with a zero-parameter state dictionary; `MULTIRES_EVENT_V2_BEST_CHECKPOINT_COLLECTIVE_CANARY_OK` is the resulting synchronization marker. It and `RELATIONAL_PRIMARY_MOUNTED_PREFLIGHT_OK` are preflight evidence only. The first training-valid marker is `MULTIRES_EVENT_V2_FORMAL_STEP2_CHECKPOINT_OK`, emitted after the same formal run completes optimizer steps 1-2 and reopens a hash-valid checkpoint. Older V2 Notebooks and `run_multires_event_v2.py` remain historical failure/verification surfaces and are not the active handoff route.

Legacy Stage A uses `train_stage_a_hour.ipynb` and `run_stage_a_hour.py`; it remains available only for historical reproduction of the textual ModernBERT experiments.
